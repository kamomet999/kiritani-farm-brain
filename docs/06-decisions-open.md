# 06 要決定事項（着手前にユーザー判断が要る）

各項目に**推奨**を添える。決まったらこのファイルに結論を追記し、該当docへ反映する。

## D1. アーキテクチャ（最重要）
本システム（写真・AI・多デバイス・増えるデータ）に耐える基盤をどれにするか。
- 選択肢：A 単一HTML / **C Next.js+Supabase+Claude API**（[03](03-architecture.md)）。
- **推奨：C**。理由＝多デバイス・写真・AI・データ蓄積を満たす唯一の現実解。無料枠で月額ほぼ¥0。
- 代替：まず極小の単一HTMLで全体図の“見え方”だけ感触を見る手もある（ただし本番は作り直しになる）。
- **決定：___**

- **推奨：まず土壌水分。可能ならECも**（肥料設計の基礎）。pH常時は鬼門なので手入力併用。
