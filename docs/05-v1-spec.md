# 05 v1 詳細仕様 — 全体図ダッシュボード ＋ 記録の土台

> v1のゴール：**イラスト化した対話型の全体図で状況が一目、タップで各ハウスの記録に入れる**。まず手入力で回し始める。
> 技術は [03](03-architecture.md) の決定に従う（推奨: Next.js+Supabase。※極小で先に感触を見る選択肢は[06](06-decisions-open.md)）。

## 画面1：全体図ダッシュボード（ホーム）

### 全体図の作り方
- 既存の航空写真マップ（[../site-map/](../site-map/)）は**位置の参照元**。表示はそのまま使わない。
- **イラストチックに新規生成した模式図**：ハウスを角丸の区画/アイコンで表現。
- **縦長を圧縮**：谷沿いの森・余白を省略し、10エリアをコンパクトに再配置（実距離でなく分かりやすさ優先）。エリアはラベルでグルーピング。
- **SVGで実装**：各ハウス=クリック可能な図形。`House.map_x/map_y` で配置。
- 状態を**色・アイコン・数値**で：
  - 作業状況：正常/要対応/遅れ（色）。未実施タスク数（数値バッジ）。
  - 生育状況：要注意フラグ（アイコン）。定植からの経過日数（数値）。
  - 測定鮮度：最終測定から日数（古ければ印）。
- 上部にサマリ（全体の 要対応/遅れ 棟数、今日やる作業数）。

### 操作
- ハウスをクリック/タップ → 画面2（詳細）へ。
- エリア絞り込み、状態フィルタ（要対応だけ 等）。

## 画面2：ハウス詳細（ドリルダウン）

選んだハウス（例 012）の細かい状態：
- **基本**：号数・エリア・定植日（編集可）・作物。
- **タイムライン**：観察・測定・作業を時系列で（過去＝日誌、未実施＝これからやる事）。
- **観察メモ入力**：日付＋テキスト（朝の知見をここに）。
- **測定入力（手入力）**：種別（土壌水分/EC/pH/地温）＋値＋方法（感覚/機器）。
- **作業入力**：作業種別＋状態（予定/完了）＋日付＋メモ。
- **道具**：このハウス/倉庫の道具（画面4と共有）。
- （v2で写真・AI指示欄を追加する余地を残す）

## 画面3：朝の入力（クイック）
- 複数ハウスをまとめて素早く記録するモード（畑を回りながら）。
- ハウスを選ぶ→観察/測定/作業をサッと入れる→保存。スマホ最適化。

## 画面4：道具（#5）
- 道具マスターの一覧・追加・編集（名前・分類・数・所在）。

## データ
- [02](02-data-model.md) の House / Area / PlantingCycle / Observation / Measurement / WorkTask / Tool を使用。
- v1では Photo / AIInstruction は**スキーマだけ用意**して未使用（v2で有効化）。
- House 60件・Area 10件・map_x/map_y は初期投入（配置は圧縮イラストに合わせて決める）。

## 受け入れ基準（Go/No-Go）
- 全体図で60棟が圧縮イラスト表示され、状態が色/アイコン/数値で読める。
- ハウスをタップ→詳細が開き、観察・測定・作業を入力→保存→再訪で残っている（多デバイスで同じデータ）。
- 朝の入力モードで複数ハウスを素早く記録できる。
- 道具の登録・閲覧ができる。
- スマホ幅で崩れない。コンソールエラーなし。

## 非スコープ（v1でやらない）
- AI作業指示の生成（v2）。写真解析（v2）。測定機の自動連携（v3）。肥料設計（v4）。

## 実装メモ
- 実装は [orchestrator-mode](../../../.claude/rules/orchestrator-mode.md) に従いCodex委譲。ホットファイル衝突を避けるためトラック分割（例: DB/スキーマ、ダッシュボードSVG、詳細フォーム）。
- 検証はClaude本体がブラウザで（v0で確立した手順）。
