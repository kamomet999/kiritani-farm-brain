# 7bay 計画書（docs）

「農場の頭脳」システムの**詳細計画**。実装前に、要件・データ・技術・段取り・v1仕様・要決定事項を docs に固める（document first）。
全体像の要約は [../CONCEPT.md](../CONCEPT.md)。ハウス住所の正本は [../site-map/house-map.md](../site-map/house-map.md)。

> ステータス: **計画フェーズ（コード未着手）**。この docs が固まってから v1 の実装に入る。

## 読む順番

| # | ドキュメント | 中身 |

| 01 | [requirements](01-requirements.md) | 何を作るか（全要素の機能要件・ユーザーストーリー・UI原則・非機能要件） |

## 原則（このプロジェクトの前提）

- **コスパ順**：安く土台になるものから。月額はできるだけ¥0に寄せる。
- **観察・測定ドリブン**：固定スケジュールでなく、現場の観察＋測定データで施策が変わる。
- **document first**：計画をdocsに書いてから実装。実装はCodexに委譲、検証は本体（Claude）が実施。
- **推測でデータを埋めない**：農園の数値・工程は未確認なら空欄/未確認と明記（[../CLAUDE.md](../CLAUDE.md)）。
