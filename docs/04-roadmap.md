# 04 ロードマップ（コスパ順）

安く土台になるものから。各フェーズは「動く価値」を1つ出す。実装はCodex委譲、検証はClaude本体。

## 依存関係の考え方

```
v1 記録土台＋全体図ダッシュボード   ← 全ての基礎（データが貯まる器＋見える化）
     ├─ v2 AI作業指示＋写真解析     ← v1にデータ入力があって初めて意味を持つ
     ├─ v3 測定機連携＋傾向表示      ← 機器選定(並行)後、v1の測定欄を自動化
     └─ v4 データ活用（肥料設計等）  ← データが貯まってから
（横断）道具の把握 … v1に同居
（並行）測定機リサーチ … 私(Claude)がv1実装と同時に進める
```

## v1 — 全体図ダッシュボード ＋ 記録の土台（＝#7＋#1）

**狙い**：見える化とデータの器を1本で。まず手入力（指の感覚メモ）から回し始める。
- イラスト化・圧縮した**対話型 全体図**（各ハウス クリックで詳細）。状態を色/アイコンで。
- ハウス詳細：定植日・日誌/作業履歴・観察メモ・測定値(手入力)・今後の作業・道具。
- 詳細は [05-v1-spec](05-v1-spec.md)。
- **Go/No-Go**：農場長が全体図で状況を把握でき、1ハウス分の記録を入れて後から辿れる。

## v2 — AI作業指示 ＋ 写真解析（＝#3＋#6）

**狙い**：朝の知見入力→明快な指示。写真→状況把握＋次の手。
- 朝、観察＋最新測定＋写真を入力 → Claudeが各ハウスの指示を生成（曖昧さ解消）。
- 生育写真をvision解析 → 所見と次の手を提示、指示に反映。
- **Go/No-Go**：「無編集で現場に出せる指示」が安定生成。写真所見が妥当。

## v3 — 測定機連携 ＋ 傾向表示（＝#2）

**狙い**：指の感覚を機器測定に置換し、手入力を自動化。
- 選定した携帯機器（土壌水分/EC等）の値を取り込み（CSV/API）→ 測定欄を自動化。
- ハウス別の測定トレンド表示。
- **前提**：測定機リサーチ（並行トラック）の結論。

## v4 — データ活用（肥料設計・資材選定）（＝#4）

**狙い**：貯まったデータから意思決定支援。
- 傾向＋作型から 追肥/防除の提案、肥料設計・資材選定の下書き。
- ※自動生成を過信せず、農場長の判断を支える位置づけ。

## 並行トラック：測定機リサーチ（私が担当）

- 素早く測れてデータ化できる携帯機器（土壌水分が核。EC/pH/地温は[06](06-decisions-open.md)で確定）。
- 「道具の把握」とも重なる。v1実装（Codex）と**同時並行**で私(Claude)が調べ、v3の入口を用意。

## マイルストーンの並び

1. docs確定（本フェーズ）→ 2. v1仕様GO → 3. Codexでv1実装 → 4. Claude検証 → 5. 実地で少し使う → 6. v2（AI）… の順。
