
# 01 要件定義

## ビジョン

桐谷の農園（ハウス 001〜061・稼働60棟）を、**観察と測定を貯めて、AIが毎朝クリアな作業指示を出す「農場の頭脳」**にする。
記録が**作業日誌**、最新データからAIが出す「次にやること」が**作業指示書**。1つで両方を兼ねる。

## 毎日のコア・ループ

```
毎朝：農場長が畑を回る
   ↓ 観察（知見）＋ 素早い測定値（土壌水分など）＋ 生育写真 を入力
AI判断層：テキスト観察＋測定値＋写真解析 を統合して状況把握
   ↓ ハウス別に「今日やること」を明快な作業指示として提示
現場：指示を見て作業 → 実施を記録（＝日誌）
   ↓ 測定値・観察・作業履歴がハウス別・時系列で蓄積
データ活用：灌水/追肥/防除の判断、肥料設計・資材選定（貯まるほど賢く）
```

## 機能要件（要素ごと）

### F0. ハウス住所マスター（✅済）
- 001〜061・10エリア（工場前/梅花/おぞれ/上野西/上野東/宮枚/公民館/垣内上/垣内中/垣内下）。004欠番・稼働60棟。
- 各ハウスに **定植日** を持たせる（生育ステージの基準）。

### F1. 記録の土台（日誌兼指示）
- ハウス別に時系列で貯める：定植日・観察メモ・測定値・実施した作業・今後やる作業。
- まず**手入力（指の感覚メモ）**から始められ、機械測定が来たら数値に置き換わる。
- ユーザーストーリー：「農場長として、各ハウスで何を・いつやったか後から辿れる」「未実施＝これからやる事として見える」。

### F2. 素早い測定 → データ化
- 圃場を指で触る感覚判断を、**携帯機器で素早く測定**してデータに残す（土壌水分が核。EC/pH/地温は要検討＝[06](06-decisions-open.md)）。
- 測定値はF1の記録に紐づく（ハウス×日時×種別×値）。

### F3. AI作業指示（曖昧さ解消）
- **朝、農場長が畑を回って得た知見を入力 → AIが各ハウスの明快な作業指示を生成**。
- 指示は「場所・作業・なぜ・どうやるか・注意」まで含み、**見れば分かる**（現場の「指示が曖昧で分からない」を解消）。
- 施策は状況で変わる前提（工程表は背景の目安）。

### F4. データ活用（意思決定支援）
- 蓄積データの傾向表示 → **肥料設計・農薬/肥料の選定**の支援。
- ※いきなり自動生成はしない。データが貯まってから段階的に。

### F5. 道具の把握
- 道具マスター（種類・数・所在）。記録土台に同居できる軽い機能。

### F6. 写真で生育解析
- 生育写真を送る → **AI(vision)が状況把握＋次の手を提案**。F3の入力モードの一つ。

### F7. 全体図ダッシュボード（★UIの主役）
- 農場全体図の上に、各ハウスの**作業状況・生育状況を色・アイコン・数値**で表示（俯瞰）。
- **クリック/タップで各ハウスの詳細**（定植日・日誌・観察・測定・写真・今後の作業）へドリルダウン。
- **全体図の作り方（重要）**：既存の航空写真マップは**位置の参照元**とし、表示は作り直す。
  - 縦長（谷沿い・森の隙間で間延び）なので、**余分な空間を省いて圧縮**。
  - **イラストチックに新規生成**（写真そのままでなく、見やすい模式図）。
  - 各パーツ（ハウス）が**クリックで動作**する対話型。

## UI原則

- **俯瞰 → ドリルダウン**：図から数値・絵・アイコンで意味が分かる → タップで細かい状態。
- 全体図がダッシュボードの中心。圧縮・イラスト化・対話型。
- 現場（スマホ）で使える。指で押しやすい。日本語。

## 非機能要件

- **コスト**：月額をできるだけ¥0に寄せる（無料枠を活用）。
- **現場利用**：スマホで入力・写真撮影・閲覧。電波が弱い圃場もあり得る（[site-map]の電波欄で要確認）。
- **多デバイス**：農場長のスマホ/PC、現場の閲覧。データは1箇所に集約したい（端末ローカル閉じ込めは不可）。
- **データ寿命**：一作（数ヶ月）〜通年でデータが増える。消えない保存が必要。
- **写真・AI**：画像保存とLLM（テキスト生成・vision）を扱える基盤が要る。
