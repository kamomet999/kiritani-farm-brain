# labor-loop — 労務ループ（作業指示→申告→進捗ボード）

桐谷のハウス（全10エリア・001〜061・稼働60棟）を対象に、
「管理者の割当 → ハウス別の明快な作業指示 → 現場が一言申告 → 進捗ボードで見える化」を回す仕組み。
一番の痛み（**パートが十分なペースで／望むやり方で作業できているか**）に直撃する本命トラック。

住所の正本は [../site-map/house-map.md](../site-map/house-map.md)（エリア名＋号数001〜061）。

---

## v0 プロトタイプの形（＝Codexに委譲する範囲）

**単一の自己完結HTMLファイル**（`index.html`）。サーバー不要・依存なし・オフライン動作・スマホ可・localStorage保存。
この環境のTLS中継/常時稼働/npm依存の壁を全部回避し、検証を最速にするため。

### 機能

1. **ハウス住所を内蔵**：10エリア・001〜061（004は欠番）をコードに埋め込む。
2. **割当入力**：管理者が本日のタスクを追加（作業内容／対象ハウス（号数複数選択 or エリア一括）／担当者／目安時間）。
3. **指示文の生成**：割当から、LINEにそのまま貼れる指示文を担当者ごとに生成（コピーボタン）。書式は [../work-instruction-system/docs/instruction-format.md](../work-instruction-system/docs/instruction-format.md) 準拠（場所・作業・理由・目安時間・注意）。
4. **進捗ボード**：割当てたハウス×状態（未着手/作業中/完了）を一覧。タップで状態更新（時刻記録）。目安時間を過ぎて未完了なら「遅れ」表示。
5. **保存/リセット**：localStorageで当日分を保存、「今日をリセット」ボタン。

### 検証（Go/No-Go）

- ブラウザで開ける／割当を数件入れてLINE用テキストが生成・コピーできる／ボードが状態更新＆再読込後も保持される。
- 「無編集でLINEに出せる指示」「ペース遅れが一目で分かる」が実感できれば、次はLINE自動連携（Messaging API＋webhook）へ。

## 位置づけ

- v0は**チャネル自動化なし**（管理者が生成テキストをLINEに手貼り、申告を見てボードを更新）。まず変換ループの価値を検証。
- 効いたら Phase 1: LINE Messaging API＋webhookで申告を自動取り込み。
