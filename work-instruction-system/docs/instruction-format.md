# 作業指示書の目標書式（Hermesスキルの種）

Hermes が `current_status.md` ＋ 管理者のラフ指示から生成すべき「本日の作業指示書」の**目標フォーマット**。
Phase 0 で品質が固まったら、この書式を Hermes のスキル（`~/.hermes/skills/`）として登録し、以後の生成に自動適用する。

---

## 設計原則（パートさんが迷わないために）

1. **専門用語を噛み砕く**（例:「飽差管理」→「窓を10cm開けて湿気を逃がす」）。
2. **場所・対象・数量を具体的に**（「A棟のミニトマト、下から3段目まで」）。
3. **手順は番号付き・1タスク1行**。判断を挟ませない。
4. **なぜやるかを一言**添える（納得して動ける／応用が効く）。
5. **所要時間の目安**を書く（段取りが立つ）。
6. **やってはいけないこと・注意**を明示。
7. 完了報告のしかたを最後に添える。

---

## テンプレート

```
📋 本日の作業指示（{日付} / {天気・気温の一言}）

■ 全体のポイント
{今日いちばん大事なこと1〜2行。例: 昨日が高温だったので水切れ注意}

── A棟：ミニトマト ──
1. {作業}：{対象・場所・数量}
   → なぜ: {理由一言}  ⏱ 目安: {分}
2. …

── B棟：{作物} ──
1. …

⚠ 注意
- {やってはいけないこと・危険・薬剤の扱いなど}

✅ 終わったら
- {完了報告のしかた。例: このLINEに「A棟おわり」と返信してください}
```

## 記入例（Phase 0 の検証用サンプル）

入力の想定:
- `current_status.md`: A棟 気温28℃/湿度75%/pH6.1、ミニトマト色づき良好・下葉に黄変あり
- 管理者のラフ指示:「A棟pH下がり気味だから液肥薄めで。色づいたの収穫して」

期待される出力:

```
📋 本日の作業指示（7/4 晴れ・朝から暑くなります）

■ 全体のポイント
今日は気温が高めです。作業はなるべく午前中に。水やりは株元にたっぷり。

── A棟：ミニトマト ──
1. 収穫：赤く色づいた実だけを摘む（オレンジは明日以降）
   → なぜ: 採り遅れると割れるため  ⏱ 目安: 40分
2. 液肥やり：いつもの半分の濃さで株元へ（1株あたりコップ1杯）
   → なぜ: 土が酸性気味なので濃い肥料を控えるため  ⏱ 目安: 30分
3. 摘葉：黄色くなった下葉を根元から取り、通路に置かず袋へ
   → なぜ: 風通しを良くし病気を防ぐため  ⏱ 目安: 20分

⚠ 注意
- まだ緑・オレンジの実は採らない
- 液肥は濃くしない（今日は薄めが正解）

✅ 終わったら
- このLINEに「A棟おわり」と返信してください。気になったこと（虫・しおれ等）も一言そえてください
```

## 品質チェック観点（Phase 0 の採点基準）

- そのまま現場に出せるか（管理者の再編集が不要か）
- 場所・対象・数量が具体的か
- 危険/禁止事項が漏れていないか
- 所要時間が現実的か
- パートさんの読解負荷が低いか（専門用語が残っていないか）
