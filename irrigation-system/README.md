# irrigation-system — 灌水・液肥 系統マップ

桐谷（高山市・〒509-4101）の谷筋に**点在する敷地**の潅水・液肥系統を、1枚に整理するための領域。
現状「どの弁がどのハウスへ行くか」「他の敷地の水源」が未整理なので、**まず現地調査で埋める**。

> 液肥出し分け問題の設計判断の経緯は
> [計画書](../../../../.claude/plans/ai-hermes-agent-iot-ph-hermes-agent-sla-federated-eclipse.md) と会話ログを参照。

---

## このマップが解く2つの用途

1. **液肥の出し分け改修の見積り素材** — 家ごとに清水/液肥を出し分けたい（濃度は変えず ON/OFF のみ）。
   方式候補：**清水本管＋液肥本管の2本管＋各ハウスに切替弁**／遠い系統は**水動力インジェクター**（電源不要）。
   ※実サイジング・機器選定・施工は灌水設計業者/メーカーに当てる。このマップは素材。
2. **将来の作業指示** — 「どの弁を開ける／切り替える」がそのままパートへの指示になる。労務ループ（LINE）と合流する。

## 現状（本人談・未整理）

- ため池小屋＋10バルブ（細かく14）。6個は小屋（手前）、4個は100m先のハウス手前。
- 液肥は小屋で**全バルブより上流**に注入 → 全系統が液肥入り＆閉じた枝にも滞留（＝混入問題）。
- **ため池系統がどの敷地を給水しているか未整理／他敷地の水源も不明。**

## 使い方

1. [survey-template.md](survey-template.md) を現地で歩いて埋める。
2. 航空写真に**敷地①②③…・バルブA/B/C…と番号を振る**（手書き・スクショ注記でOK）。
3. 距離は**目測/歩測でOK**（正確値は後で業者が実測）。
4. **分からない欄は空欄のまま**。推測で埋めない（[../CLAUDE.md](../CLAUDE.md) のデータ方針）。
5. 埋まったら、系統ごとに「2本管化 or 現地インジェクター」の当たりを一緒に付ける。
